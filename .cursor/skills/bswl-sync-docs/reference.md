# API docs sync reference

## Source-of-truth files (buildingswell)

### Route inventory

| File | Purpose |
|------|---------|
| `packages/server/src/routes/v2.routes.ts` | Mount table for `/api/v2/*` |
| `packages/server/src/v2/core/api/factory/index.ts` | Standard CRUD generated by `buildModelRouter` |
| `packages/server/src/v2/features/*/router.ts` | Custom endpoints per resource |

### Schemas and domain metadata

| File | Purpose |
|------|---------|
| `packages/server/src/v2/features/*/model.ts` | Zod `schema`, `createSchema` per resource |
| `packages/server/src/v2/core/model/bswl-entity-model.ts` | Shared entity fields |
| `packages/server/src/v2/core/model/index.ts` | Base columns (`id`, `organizationId`, timestamps) |
| `packages/shared/src/v2/deliverable/concepts/registry.ts` | Concept registry DTO |
| `packages/shared/src/v2/deliverable/` | `ProgressActionSchema`, `ScheduleActionSchema` |

### Query semantics

| File | Purpose |
|------|---------|
| `packages/server/src/v2/core/api/factory/utils.ts` | `filter`, `orderBy`, `select`, joins |
| `packages/server/src/v2/core/api/filter-parser/` | Filter grammar |

## Public vs internal mounts

Documented in this site (`PUBLIC_MOUNTS` in the extract script):

```
deliverable, project, comment, notification,
qc-inspection-set, qc-template, stage, document, note,
timesheet, worker, role, organization-member, api-key, concepts
```

**Internal** — mounted in `v2.routes.ts` but out of scope for public docs:

```
insights, form-triggers, page-view, category,
dashboard, system-log, labs
```

## buildModelRouter expansion

Given `buildModelRouter({ readOnly, enableImportExport, enableJsonImportExport })`:

### Always registered

| Method | Path |
|--------|------|
| GET | `/` |
| POST | `/query` |
| GET | `/:id` |
| GET | `/count` |
| POST | `/count` |
| POST | `/group-and-count` |

### When `readOnly: false` (default)

| Method | Path | Notes |
|--------|------|-------|
| POST | `/` | Create one or array |
| PATCH | `/bulk` | Bulk update |
| PATCH | `/:id` | Update |
| PUT | `/bulk` | Bulk upsert |
| PUT | `/:id` | Upsert |
| DELETE | `/:id` | Omitted for worker (hard delete forbidden) |
| GET | `/export` | When `enableImportExport: true` |
| POST | `/import` | When `enableImportExport: true` |
| GET | `/export/json` | When `enableJsonImportExport: true` |
| POST | `/import/json` | When `enableJsonImportExport: true` |

`DELETE /bulk` is never registered (globally forbidden).

### readOnly resources

`timesheet` and insight models use hand-rolled or `readOnly: true` routers — only read paths above.

## Intentional doc omissions

Code may register routes that are deliberately excluded from public docs:

| Endpoint | Reason |
|----------|--------|
| `DELETE /worker/{id}` | Hard delete forbidden; archive via PATCH |
| `POST/PATCH/PUT/DELETE /notification/*` (writes) | Notifications are system-generated |
| `POST/DELETE /role/test` | Internal impersonation helper |
| `GET /qc-template/by-template` | Internal lookup |
| `GET/POST /timesheet/count` | Not exposed in public docs |
| Org-member import/export, bulk, delete | Admin UI only |
| Most `api-key` write factory routes | Custom POST/PATCH handlers replace factory create/update |

When the extract script flags these, do not add them unless product asks to document them.

## OpenAPI conventions in this repo

- Server: `https://{instance}/api/v2` — defaults to `app.buildingswell.com` in the API playground (editable)
- Auth: `X-API-Key` header (`ApiKeyAuth` security scheme)
- Path params: `{id}`, `{rootId}` (OpenAPI style, not Express `:id`)
- List endpoints use shared `components/parameters` refs
- Quote YAML strings containing colons

## docs.json API reference groups

Groups under the `"API reference"` tab must list every public path as `"METHOD /path"` strings. When adding a new OpenAPI path, add the matching nav entry in the appropriate group (or create a new group).
